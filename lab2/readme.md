#  Складской учёт товаров


**Приложение для ведения складского учёта** с удобным графическим интерфейсом на **Python** и **tkinter**.  
Поддерживает полный CRUD, постраничный просмотр, поиск с автоподстановкой, массовое удаление по условиям и импорт/экспорт в XML.

---

## 📌 Основные возможности

| Функция | Описание |
|---------|----------|
| **CRUD операции** | Добавление, редактирование, удаление товаров с валидацией полей |
| **Пагинация** | Настраиваемое количество записей на страницу (5/10/20/50) |
| **Умный поиск** | Автоматическая фильтрация по названию, количеству, производителю, УНП, адресу |
| **Массовое удаление** | Поиск по условиям, предпросмотр, множественный выбор, подтверждение |
| **Импорт/экспорт XML** | Загрузка (SAX) и сохранение (DOM) данных в XML-файлы |
| **Современный дизайн** | Нежно-зелёная цветовая схема интерфейса |
| **Демо-данные** | Генератор 100 тестовых записей (компьютеры + обувь) |

---

##  Сценарии демонстрации

### 1️⃣ Добавление нового товара

**Шаги:**

1. Нажмите кнопку **«➕ Добавить»** 

2. Заполните поля в открывшемся диалоговом окне:

| Поле | Пример ввода |
|------|--------------|
| Название товара | `Смартфон Xiaomi Note 11` |
| Производитель | `Xiaomi` |
| УНП производителя | `123456789` |
| Количество на складе | `25` |
| Адрес склада | `ул. Советская, 15, склад B2, ряд 3` |

3. Нажмите кнопку **OK**

**Ожидаемый результат:**

- Новый товар появляется в таблице
- В строке состояния отображается сообщение: `"Товар 'Смартфон Xiaomi Note 11' добавлен"`

---

### 2️⃣ Редактирование товара

**Шаги:**

1. Выберите любую строку в таблице (кликните мышью)
2. Нажмите кнопку **«✏️ Редактировать»** 
3. В открывшемся диалоге измените поле **Количество** на `100`
4. Нажмите кнопку **OK**

**Ожидаемый результат:**

- Данные в таблице обновляются
- В строке состояния отображается сообщение об успешном обновлении

---

### 3️⃣ Поиск с автоматической фильтрацией

**Шаги:**

1. Нажмите кнопку **«🔍 Найти»** 
2. В открывшемся диалоге **«Поиск товаров»** введите условия:

| Поле | Значение |
|------|----------|
| Название | `Ноутбук` |
| Производитель | `Lenovo` |

3. Фильтр применится **автоматически** после ввода
4. Для сброса фильтров нажмите **«🔄 Сбросить все фильтры»**

**Ожидаемый результат:**

- Таблица показывает только ноутбуки Lenovo
- В заголовке окна отображается: `"Поиск товаров - найдено: 3"`
- После сброса фильтров таблица показывает все записи

---

### 4️⃣ Массовое удаление по условиям

**Шаги:**

1. Нажмите **«🗑️ Удалить по условиям»** 
2. В диалоге **«Удаление товаров по условиям»** задайте фильтр:

| Поле | Значение |
|------|----------|
| Количество | `0` |

3. Таблица покажет подходящие записи
4. Выберите строки 
5. Нажмите красную кнопку **«🗑️ УДАЛИТЬ ВЫБРАННЫЕ ЗАПИСИ»**
6. Подтвердите удаление

**Ожидаемый результат:**

- Появляется сообщение: `"✅ Успешно удалено N записей"`
- Выбранные записи исчезают из таблицы
- Главная таблица обновляется

---

### 5️⃣ Сохранение и загрузка XML

**Сохранение:**

1. Нажмите **«💾 Сохранить»** 
2. Выберите имя файла: `my_warehouse.xml`
3. Нажмите **Сохранить**

**Ожидаемый результат:** `"Данные успешно сохранены"`

**Загрузка:**

1. Нажмите **«📂 Загрузить»** 
2. Выберите файл: `computers_50.xml`
3. Нажмите **Открыть**

**Ожидаемый результат:** Все 50 записей загружены, таблица обновлена

---

### 6️⃣ Пагинация

**Шаги:**

1. Загрузите `computers_50.xml` (50 записей)
2. Используйте кнопки навигации:

| Кнопка | Действие |
|--------|----------|
| `<<` | Первая страница |
| `<` | Предыдущая страница |
| `>` | Следующая страница |
| `>>` | Последняя страница |

3. Измените **«Записей на странице»** с `10` на `20`

**Ожидаемый результат:**

- При 10 записях: 5 страниц `(1/5)`
- При 20 записях: 3 страницы `(1/3)`
- Корректное отображение номера страницы
